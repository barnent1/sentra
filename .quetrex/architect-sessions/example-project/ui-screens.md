## Screen: Dashboard

### Route
`/dashboard`

### Description
Main dashboard showing user's bookmarks in grid layout with quick actions.

### On Load
- Fetch user's bookmarks from API
- Show skeleton loading state
- Load user preferences (view mode, sort order)

### User Actions
- **Click quick add button**: Opens QuickAddModal for adding new bookmark
- **Click bookmark card**: Navigates to bookmark detail page
- **Click filter dropdown**: Opens filter menu with tags and categories
- **Toggle view mode**: Switches between grid and list view

### States
- Empty state: No bookmarks yet
- Loading state: Skeleton cards
- Error state: Failed to load bookmarks
- Populated state: Bookmarks displayed

### Validation
- User must be authenticated
- Bookmarks must belong to current user

### Error Handling
- Network error: Show retry button
- Empty results: Show onboarding message
- Permission error: Redirect to login

### E2E Tests
- **User adds first bookmark**:
  1. Navigate to /dashboard
  2. See empty state message
  3. Click quick add button
  4. Paste URL in modal
  5. Click save
  6. Verify bookmark appears in grid

- **User filters bookmarks by tag**:
  1. Navigate to /dashboard
  2. Click filter dropdown
  3. Select "Work" tag
  4. Verify only work bookmarks shown

## Screen: BookmarkDetail

### Route
`/bookmarks/:id`

### Description
Detailed view of a single bookmark with notes, tags, and actions.

### On Load
- Fetch bookmark details by ID
- Fetch related bookmarks
- Show loading skeleton

### User Actions
- **Click edit button**: Opens edit modal
- **Click delete button**: Shows confirmation dialog
- **Add tag**: Opens tag selector
- **Add note**: Opens note editor

### States
- Loading state: Skeleton UI
- Loaded state: Full bookmark details
- Error state: Bookmark not found

### Validation
- Bookmark ID must be valid UUID
- User must own the bookmark

### Error Handling
- Not found: Show 404 page
- Permission denied: Redirect to dashboard
- Network error: Show retry button

### E2E Tests
- **User views bookmark details**:
  1. Navigate to /dashboard
  2. Click on bookmark card
  3. Verify bookmark details displayed
  4. Verify related bookmarks shown

- **User edits bookmark**:
  1. Navigate to /bookmarks/123
  2. Click edit button
  3. Update title
  4. Click save
  5. Verify changes reflected

## Screen: Settings

### Route
`/settings`

### Description
User settings page with preferences, account, and integrations.

### On Load
- Fetch user settings
- Fetch connected integrations
- Show current plan

### User Actions
- **Update theme**: Changes color scheme (light/dark/auto)
- **Update notification preferences**: Toggle email notifications
- **Connect integration**: Opens OAuth flow for external service
- **Change password**: Opens password change modal

### States
- Loading state: Skeleton form
- Loaded state: Editable settings
- Saving state: Disabled inputs with spinner

### Validation
- Email must be valid format
- Password must meet requirements (8+ chars, special char)

### Error Handling
- Save failed: Show error toast with retry
- OAuth failed: Show error message with support link

### E2E Tests
- **User changes theme**:
  1. Navigate to /settings
  2. Click theme dropdown
  3. Select "Dark"
  4. Verify theme applied immediately
  5. Refresh page
  6. Verify theme persisted

- **User connects Google Drive**:
  1. Navigate to /settings
  2. Scroll to integrations
  3. Click "Connect Google Drive"
  4. Complete OAuth flow
  5. Verify integration shown as connected
